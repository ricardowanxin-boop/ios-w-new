# Goal
Identify launch blockers and produce a release-ready QA and submission checklist.

# Workflow
1. Review core flows.
2. Review onboarding and first-run experience.
3. Review save/edit/delete behavior.
4. Review notifications, settings, and permissions.
5. Review paywall/purchase flow.
6. Produce blockers and improvements.
7. Prepare App Store readiness checklist.

# Output files
- /qa/test-cases/YYYY-MM-DD-test-plan.md
- /qa/bug-reports/YYYY-MM-DD-release-risks.md
- /launch/app-store/YYYY-MM-DD-submission-checklist.md

# Standards
- prioritize trust-sensitive flows
- separate blockers from nice-to-have improvements
- keep findings actionable
